Versión 2.12.0

Este es un lanzamiento de mejora y corrección de errores, ahora se requiere versión 11 de Java. Como el repositorio principal de zaproxy/zaproxy ha alcanzado 10 000 estrellas, lo llamamos el lanzamiento de las "Diez mil estrellas".

This release fixes an HTML Injection vulnerability in the ZAP Desktop which was rated a P3 / Medium level vulnerability. While we do not think that it can be exploited in any meaningful way, desktop users are still recommended to update from older ZAP versions a.s.a.p.

Estas notas de versión no incluyen todos los cambios incluidos en los complementos actualizados desde 2.11.1.

Algunas de las mejoras más significativas:

Complemento de red

Se ha cambiado el código de red principal por un nuevo complemento, lo que significa que los cambios ya no están vinculados a las versiones principales/estables. Este complemento utiliza una pila de red moderna que hará que sea mucho más fácil admitir protocolos modernos como HTTP/2
Además, se han añadido las siguientes funciones:
  • Alias de servidor proxy/local
  • El Proxy y el API de ZAP ya no están vinculados.
  • HTTPS pass-through
  • Configuración del período de validez del certificado

Complemento Spider

Para facilitar mejoras funcionales y correcciones de errores, el núcleo de Spider se ha movido a un complemento, lo que significa que dichos cambios ya no están vinculados a las versiones principales/estables. Otros complementos que utilizan las funciones de Spider tradicional también se han rediseñado para admitir el complemento Spider, incluidos: Quick Start, Form Handler, GraphQL, OpenAPI, SOAP y Automation Framework. Más detalles a continuación.

Complemento para importar/exportar

El complemento Importar/Exportar permite importar/exportar datos (por ejemplo, mensajes HTTP, URL) a/desde ZAP, reemplaza la funcionalidad principal y los siguientes complementos que ya no estarán disponibles en el mercado:
  • Importar un archivo que contiene las URL
  • Importar Archivo de Log
  • Guardar Mensaje Raw
  • Guardar Mensaje XML

Complemento de Base de Datos

Se introdujo un nuevo complemento para la funcionalidad relacionada con la base de datos en ZAP. Este complemento proporciona el motor de base de datos SQLite para que lo usen otros complementos. También agrega soporte para la base de datos permanente ZAP.

La base de datos permanente permite almacenar información que se puede utilizar en las sesiones de ZAP. Por ejemplo, el complemento OAST lo utiliza para conservar las cargas útiles de BOAST que se pueden sondear. en futuras sesiones de ZAP para enumerar las interacciones fuera de banda realizada en el servicio mientras ZAP no se estaba ejecutando.

Escáner Pasivo Multi-Hilo

El escáner pasivo se actualizó para usar una cantidad configurable de subprocesos, por defecto 4. Se ha demostrado que esto reduce significativamente el tiempo dedicado al procesamiento de la cola de exploración pasiva.

Eliminada la Telemetría por Bit.ly

A partir de esta versión, ZAP ya no usará bit.ly para ninguna telemetría. En su lugar, utilizaremos nuestros propios servicios en el dominio zaproxy.org. Para obtener detalles completos, consulte las preguntas frecuentes: ¿Qué 'llamadas a casa' hace ZAP?.

ZAP API Breaking Changes:

VIEW alert / alertCountsByRisk

The returned data now includes the False Positive count. This change may break existing consumers as the number of expected alerts might no longer be the same. For example, if a Medium risk alert is marked as False Positive, the structure of returned data will be: 

{"High":0,"Low":3,"Medium":0,"Informational":2,"False Positive":1}
instead of: 

{"High":0,"Low":3,"Medium":1,"Informational":2}

Endpoints With Response Changes

The following endpoints used to return "OK" for all inputs. They now return suitable error messages (such as "does_not_exist" or "illegal_parameter") when the inputs are invalid.
  • ACTION ascan / enableScanners
  • ACTION ascan / disableScanners
  • ACTION ascan / setEnabledPolicies
  • ACTION pscan / enableScanners
  • ACTION pscan / disableScanners

Nuevos Endpoints API para ZAP:

  • ACTION pscan / clearQueue
