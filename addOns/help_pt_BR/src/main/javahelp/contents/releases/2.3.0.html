Versão 2.3.0

As seguintes alterações foram feitas nesta versão:

Mudanças significativas:

ZAP versão 'lite'

Para este release estamos fornecendo uma versão 'lite' do ZAP, além da ‘full’ . Ela contém exatamente o mesmo código do núcleo, mas tem menos Complementos. Claro que depois você pode baixar do Marketplace todos os complementos 'ausentes', fazendo assim upgrade para o ZAP 'full'.
A versão 'lite' é destinada a pessoas novas em segurança que precisam de menos funcionalidade inicial e que tenham, assim esperamos, mais facilidade para começar. Também será apropriada para pessoas à procura de um download mais rápido ou para aquelas que desejam personalizar com complementos que escolham.

Suporte a eventos do lado do cliente (navegador)

Você agora pode ver, interceptar, manipular, reenviar e fazer fuzzing de eventos no lado do cliente. Isso inclui postMessages, de modo que você pode agora detectar vulnerabilidades XSS naseadas em DOM nos postMessages. Esta é a primeira fase de uma série de alterações planejadas para suportar os testes de aplicativos AJAX e HTML5 ainda mais eficazmente.

Suporte de autenticação melhorado

O suporte do ZAP para autenticação foi completamente renovado, para lidar facilmente com tipos complexos de cenários e métodos de autenticação. Também foi adicionado suporte para os scripts definidos pelo usuário que permitem a você manipular esquemas de autenticação personalizada. Além disso, agora ZAP entende e permite que você configure os usuários dos aplicativos da Web Então várias ações do ZAP podem ser executadas do ponto de vista de usuários definidos. Para começar, confira os novos painéis Autenticação e Usuários nas propriedades de sessão de cada um dos contextos definidos.

Suporte para aplicaçoes não-padronizadas

Esta versão inclui suporte para aplicações de 'página única' e separadores de chave/valor não-padrão. Agora você pode controlar essas configurações através do novo painel Estrutura nas propriedades da sessão.

Novos vetores de entrada, incluindo scripts definidos pelo usuário

ZAP suporta novas opções para definir vetores de entrada ou seja, os elementos de uma request que ZAP usará para atacar. As novas opções estão disponíveis no painel Vetores de Entrada da varredura ativa em opções. Também foi adicionado suporte para a definição de scripts personalizados que definem novos vetores de entrada.

Política de varredura - controle detalhado

A política de varredura agora tem um controle detalhado, permitindo a você ajustar as regras individuais de varredura. Você também pode definir, carregar e salvar políticas de varredura, para manter um conjunto de políticas que funcionam bem em circunstâncias diferentes.
Além disso, por padrão ZAP agora varrerá parâmetros de serviço conhecidos (por exemplo, ViewState) acelerando o processo geral de varredura. Isso é completamente configurável pelo usuário, permitindo-lhe especificar exatamente quais parâmetros o ZAP deve ignorar.

Diálogo avançado de Varredura Ativa

Um novo diálogo de 'Varredura Ativa Avançada' permite que você especifique exatamente como você quer que o scanner ativo funcione. Ele permite a você especificar 'vetores personalizados' que explicitamente definem quais string você quer para atacar. Ele também suporta a opção para escanear como qualquer um dos usuários que você definiu para o aplicativo sob teste. Inicie uma 'Varredura Ativa Avançada' via menu de Ferramentas ou via seção de Ataque no menu do botão direito.

Opções estendidas de linha de comando

Agora você pode executar ZAP 'inline', ou seja, sem iniciar a interface do usuário do ZAP ou um daemon. Neste modo, você pode executar ataques simples ou executar scripts que podem acessar toda a funcionalidade do ZAP. Você agora também pode substituir qualquer uma das opções definidas no arquivo de configuração através de parâmetros inseridos pela linha de comando.

Mais suporte à API

A API foi estendida para apoiar ainda mais as funcionalidades do ZAP.

Arquivo de help internacionalizado

O arquivo de ajuda foi internacionalizado e está em processo de tradução para muitos outros idiomas via https://crowdin.com/project/zap-help. Se você usar o ZAP em um dos vários idiomas que oferecemos, os arquivos de ajuda incluirão todas as traduções disponíveis para aquele idioma, voltando ao inglês para as frases que ainda não foram traduzidas.
Os idiomas com uma quantidade significativa de páginas de ajuda traduzidas incluem:
  • Português do Brasil
  • Francês
  • Japonês
  • Espanhol

Atalhos de teclado

Todos os itens de menu agora podem ser invocados através de atalhos de teclado. Há atalhos são definidos para praticamente todos os casos, mas você pode configurar suas preferências no painel de opções do Teclado painel das opções.

Novas opções de interface do usuário

Há uma nova opção para alterar a exibição, para que a aba selecionada ocupe a tela toda. Isso é útil usando o ZAP em telas pequenas. Há também uma opção para alternar a visibilidade dos nomes das abas em ligado e desligado para economizar mais espaço.

A maioria das listas na interface do usuário também foram convertida em tabelas, que permitem a você alterar as larguras de coluna e definir exatamente quais colunas são exibidas e como as tabelas são classificadas.

Mais funcionalidades para os Complementos

Mais um pouco da funcionalidade do núcleo do ZAM foi movida para os complementos, o que nos permite entregar atualizações dinamicamente via ZAP Marketplace, em vez de exigir novos releases completos.
This includes the language packs, so translations made to the ZAP UI via https://crowdin.com/project/zaproxy can be downloaded within ZAP or even automatically installed.

Regras de varredura ativa e passiva novas e melhoradas

Many of the release status active and passive scanning rules have been improved. There are new alpha and beta status rules and many rules have been promoted from alpha to beta and from beta to release status.

Outras variadas alterações e acréscimos

  • Uma nova opção de parar as regras de verificação individualmente sem parar a varredura toda
  • Um novo botão na barra de ferramentas que permite a você rapidamente e facilmente gravar scripts do Zest.
  • Foi criado um novo grupo para partilha de scripts ZAP (https://groups.google.com/group/zaproxy-scripts).
  • Capacidade de fazer spider de aplicações com base em metadados de controle de fonte (SVN e Git) expostos através de um servidor web
  • Capacidade de forçar interrupções de dentro de scripts de Proxy
