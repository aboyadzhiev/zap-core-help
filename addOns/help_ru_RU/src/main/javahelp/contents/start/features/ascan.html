Активное Сканирование

Активное сканирование пытается найти потенциальные уязвимости с помощью известные атаки на выбранные цели.

Активное сканирование - это атака на эти цели. <0 /> Вы НЕ должны использовать его в веб-приложениях, которыми вы не владеете.

Чтобы облегчить идентификацию ZAP-трафика и исключений брандмауэра веб-приложений, ZAP сопровождается скриптом "AddZapHeader.js", который можно использовать для добавления определенного заголовка ко всему проходящему через него трафику. или происходит из ZAP. например: Инициатор X-ZAP: 3

Следует отметить, что активное сканирование позволяет находить только определенные типы уязвимостей. <0 /> Логические уязвимости, такие как нарушение контроля доступа, не будут обнаружены любое активное или автоматическое сканирование уязвимостей. <0 /> В дополнение к активному тестированию на проникновение всегда следует проводить ручное тестирование на проникновение. сканирование для поиска всех типов уязвимостей.

Активное сканирование настраивается с помощью Параметры экрана активного сканирования .
Выполняемые правила настраиваются с помощью политик сканирования - их можно сколько угодно.
