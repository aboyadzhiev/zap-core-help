Varredura Passiva

O ZAP verifica passivamente, por padrão, todas as mensagens HTTP (pedidos e respostas), enviadas para o aplicativo da web que está sendo testado.
Varredura passiva não altera os pedidos, nem as respostas de forma alguma e, portanto, é seguro para uso.
A varredura é executada em um thread de segundo plano para garantir que isso não diminua a exploração de um aplicativo.

O comportamento (principal) do scanner passivo pode ser configurado usando a Tela de opções de Scanner passivo.

Varredura passiva também pode ser usada para automaticamente adicionar tags e criar alertas para potenciais problemas.
Um conjunto de regras para a marcação automática é fornecida por padrão. Elas podem ser alteradas, excluídas ou adicionadas através da tela de opções de Tags de varredura passiva.

The alerts raised by passive scan rules can be configured using the Options Passive Scan Rules screen.
