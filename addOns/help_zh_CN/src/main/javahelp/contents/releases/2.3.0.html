2.3.0版本

此版本中进行了以下更改：

重大变化：

ZAP ‘lite’ 版本

在这个版本中，我们提供了一个叫做 ‘lite’ 的版本，该版本不同于 ‘full’ 版本。 这个‘lite’版本包含与‘full’版本完全相同的核心代码，但只包含了更少的默认附加组件。 当然，您可以从ZAP市场下载所有“缺失”的附加组件，将‘lite’版本‘升级’到完整版本。
这个 ‘lite’ 版本的目标人群是刚刚接触网络安全问题的用户，他们需要较少的初始功能，希望能够更容易地开始使用ZAP。 它也适用于寻找较小下载包的用户，或者那些希望自定义安装附加组件的用户。

支持客户端（浏览器）事件

您现在可以查看，截取，操纵，重新发送和模糊化 客户端事件。 这包括postMessages（Windows API 中的一个常用函数），所以您现在可以在postMessages中检测到基于DOM的XSS漏洞。 这是一系列计划变更中的第一阶段，以便更有效地支持AJAX和HTML5应用程序的测试。

增强的身份验证支持

ZAP对身份验证的支持已经完全修改，以便可以轻松地处理复杂类型的 身份验证方法和方案。还为用户自定义的脚本添加了支持，以便允许您处理自定义身份验证方案。 另外，现在ZAP明白并允许您配置 web应用程序的用户 所以整个ZAP的各种操作可以从定义的用户的角度来执行。请检查新的 身份验证 和 用户 面板（在每个定义的上下文的会话属性中）。

支持非标准应用程序

这个版本包括支持 ‘单页’应用程序 和 非标准键值分隔符。您现在可以控制这些设置通过新的 结构 面板（在会话属性中）。

新的输入向量包括用户定义的脚本

ZAP支持新的选项来定义 输入向量 例如，ZAP将要攻击的请求的元素。新选项在 主动扫描输入向量 选项面板中。还增加了对定义新输入向量的自定义脚本的支持。

扫描策略 - 细粒度（fine grained）控住

扫描策略现在有一个 细粒度（fine grained）控制, 允许您调整单个扫描器的规则。您还可以定义，加载和保存扫描策略，使您能够维护一套在不同情况下都能运行良好的策略。
另外，在默认情况下，ZAP将不会扫描众所周知的服务参数（例如，_VIEWSTATE），从而加速整个扫描过程。 这完全是用户可配置的，允许您指定ZAP应该忽略的参数。

高级主动扫描对话框

一个新的 高级主动扫描 对话框允许您指定您希望主动扫描器的功能。 它允许您指定 ‘自定义向量’ （明确定义您想要攻击的字符串）。它还支持您为被测试的应用程序定义的任何用户的扫描选项。 高级主动扫描 通过该 工具 菜单或通过 攻击 （右键单击弹出菜单的部分）。

扩展命令行选项

您现在可以运行ZAP的‘inline’模式，例如， 不启动ZAP用户界面或守护进程. 在这种模式下，您可以运行简单的攻击或运行可以访问所有ZAP功能的脚本。 现在您也可以通过命令行参数覆盖配置文件中已经定义的任何选项。

更多API的支持

API已经扩展到支持更多的ZAP功能。

国际化的帮助文档

该帮助文档已被国际化，而且它正在被翻译成许多其他语言的过程中： https://crowdin.com/project/zap-help。 如果您使用我们支持的多种语言之一的ZAP，那么帮助文档将包含该语言的所有可用翻译，而对于尚未翻译的短语则默认为英语显示。
具有大量已经翻译的帮助页面的语言包括：
  • 波斯尼亚语
  • 法语
  • 日语
  • 西班牙语

键盘快捷键

所有菜单项现在可以通过键盘快捷键调用。 默认的是所有情形都是被定义的，但您可以配置自己的首选项通过 键盘 选项面板。

新的UI选项

有一个新的选项来更改显示，以便选中的选项卡占用 全屏模式。这在小屏幕上使用ZAP时很有用。还有一个选项可以选择 切换选项卡名称的可见性 以便进一步节省空间。

大多数UI列表也被转换为表格，这些表格允许您更改每一列的宽度并准确定义要显示的列以及表格的排序方式。

更多的功能移动到附加组件

更多的核心功能已经被转移到附加组件中，允许我们通过ZAP 市场动态地提供更新，而不需要新的完整版本。
This includes the language packs, so translations made to the ZAP UI via https://crowdin.com/project/zaproxy can be downloaded within ZAP or even automatically installed.

新的和改进的主动和被动扫描规则

Many of the release status active and passive scanning rules have been improved. There are new alpha and beta status rules and many rules have been promoted from alpha to beta and from beta to release status.

其他各种变化和添加

  • 在不停止整个扫描的情况下停止单个扫描规则的新选项
  • 一个新的工具栏按钮，使您可以快速，方便地记录Zest脚本。
  • 已创建一个用于共享 ZAP 脚本 ( https://groups.google.com/group/zaproxy-scripts ) 的新组。
  • 基于通过Web服务器公开的源代码管理元数据（SVN和Git）的Spider应用程序的能力
  • 从代理脚本中强制中断的能力
