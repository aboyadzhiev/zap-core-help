Окно параметров JVM

Этот экран позволяет настроить параметры JVM, используемые при запуске ZAP.

Параметры JVM

Текстовое поле произвольного формата, которое будет добавлено к вызову командной строки Java. при вызове ZAP через zap.sh или zap.bat.

Опция была добавлена, чтобы можно было установить максимальный размер пула выделения памяти Java, который имеет форму: -Xmx n где n это размер в байтах.
Хорошими значениями для этого поля могут быть:

  • -Xmx256m
  • -Xmx512m
  • -Xmx1024m

В отличие от других параметров ZAP, они хранятся в файле. .ZAP_JVM.properties в каталоге ZAP пользователя по умолчанию, что зависит от используемой ОС, например:

  • Windows 7/8: C:\Users\<username>\ZAP
  • Windows XP: C:\Documents and Settings\<username>\ZAP
  • Linux: ~/.ZAP
  • Mac OS: ~/Library/Application Support/ZAP
Если вы допустили ошибку при установке этих параметров, ZAP может не запуститься.
Если это произойдет, удаление этого файла должно решить проблему.

Примечания для Windows: Эти параметры не используются при запуске ZAP через исполняемый файл. Каталог, в котором находится файл .ZAP_JVM.properties находится в зависимости от переменной окружения %USERPROFILE%.

Сведения о памяти JVM

В нижней части диалогового окна отображаются сведения о текущей конфигурации памяти JVM. В том числе:
  • Размер (Size) — объем памяти, выделенный в данный момент для JVM.
  • Используемый - объем используемой в данный момент выделенной памяти (не свободной).
  • Max — объем памяти, который может занимать JVM.
Значения отображаются в удобочитаемых единицах IEC на основе расчета по основанию 2 (например: 1024 байта равны 1 КБ и т. д.).

Обычно значение max должно быть близко к -Xmx (если указано). Однако при тестировании под Java 8 в ОС Windows наблюдалось следующее поведение.

Было указано -Xmx512m (которое должно быть рассчитано примерно до 488 МиБ), однако отображалось 455,5 МиБ.
-Xms512m -Xmx512m был указан, однако отображалось 475МиБ.
-XX:+AlwaysPreTouch -Xms512m -Xmx512m и отображалось 474,5 МБ.

Таким образом, кажется, что JVM выполняет некоторые вычисления/манипуляции при выполнении, что может означать, что результаты могут не соответствовать ожидаемому для варианты ввода пользователя (это проблема JVM, а не проблема ZAP).
