выпуск 2.3.0

В этот выпуск внесены следующие изменения:

Существенные изменения:

ZAP "облегченная" версия

Для этого выпуска мы предоставляем «Облегченную» версию ZAP в дополнение к 'полной' ' версии Он содержит точно такой же основной код, но включает меньше надстроек по умолчанию. Конечно, вы можете загрузить все «недостающие» надстройки с торговой площадки ZAP, чтобы «обновить» облегченную версию до полной.
«Облегченная» версия предназначена для людей, плохо знакомых с безопасностью, которым требуется меньше начальных функций, с которыми, будем надеяться, будет проще начать работу. Она также подойдет тем, кто хочет скачать меньший объем или желает точно настроить, какие надстройки они устанавливают.

Поддержка клиентских (браузерных) событий

Теперь вы можете просматривать, перехватывать, манипулировать, повторно отправлять и фаззинг клиентские события . Сюда входят сообщения postMessages, поэтому теперь вы можете обнаруживать уязвимости XSS на основе DOM в сообщениях postMessages. Это первая фаза в серии запланированных изменений для еще более эффективной поддержки тестирования приложений AJAX и HTML5.

Поддержка расширенной аутентификации

Поддержка аутентификации ZAP была полностью переработана, чтобы упростить обработку сложных типов методы и сценарии аутентификации . Также была добавлена поддержка пользовательских сценариев, которые позволяют обрабатывать настраиваемые схемы аутентификации. Кроме того, теперь ZAP понимает и позволяет настраивать Пользователи веб-приложений поэтому различные действия в ZAP могут выполняться с точки зрения определенных пользователей. Для начала ознакомьтесь с новым Аутентификация и Пользователи панели в свойствах сеанса для каждого из определенных контекстов.

Поддержка нестандартных приложений

Этот выпуск включает поддержку "Одностраничные" приложения и нестандартные разделители "ключ-значение" . Теперь вы можете управлять этими настройками с помощью нового Структура панель в свойствах сеанса.

Новые входные векторы, включая пользовательские скрипты

ZAP поддерживает новые параметры для определения входные векторы то есть элементы запроса, которые будет атаковать ZAP. Новые параметры доступны в Входные векторы активного сканирования панель опций. Также была добавлена поддержка для определения пользовательских сценариев, которые определяют новые векторы ввода.

Политика сканирования - детальный контроль

Политика сканирования теперь имеет мелкозернистый контроль , позволяя настраивать отдельные правила сканера. Вы также можете определять, загружать и сохранять политики сканирования, что позволяет поддерживать набор политик, которые хорошо работают в различных обстоятельствах.
Кроме того, по умолчанию ZAP теперь не будет сканировать известные параметры службы (например, __VIEWSTATE), что ускоряет общий процесс сканирования. Это полностью настраивается пользователем, что позволяет вам точно указать, какие параметры ZAP должен игнорировать.

Диалог расширенного активного сканирования

Новый Диалог расширенного активного сканирования диалоговое окно позволяет точно указать, как должен функционировать активный сканер. Это позволяет указать "Пользовательские векторы" это явно определяет, какие строки вы хотите атаковать. Он также поддерживает возможность сканирования как любой из пользователей, определенных для тестируемого приложения. Начать Расширенное активное сканирование через Инструменты меню или через Атака раздел контекстного меню, вызываемого щелчком правой кнопки мыши.

Расширенные параметры командной строки

Теперь вы можете запускать ZAP "inline", т.е. без запуска ZAP UI или демона . В этом режиме вы можете запускать простые атаки или запускать сценарии, которые могут получить доступ ко всем функциям ZAP. Теперь вы также можете переопределить любые параметры, определенные в файле конфигурации, с помощью параметров командной строки.

Дополнительная поддержка API

API был расширен для поддержки еще большей функциональности ZAP.

Internationalized help file

Файл справки был интернационализирован и находится в процессе перевода на многие другие языки через https://crowdin.com/project/zap-help. Если вы используете ZAP на одном из многих языков, которые мы поддерживаем, то файлы справки будут включать все доступные переводы для этого языка, в то время как по умолчанию обратно на английский язык для фраз, которые еще не были переведены.
Языки со значительным количеством переведенных справочных страниц включают:
  • Боснийский
  • Французский
  • Японский
  • Испанский

Горячие клавиши

Все пункты меню теперь можно вызывать с помощью горячих клавиш. Значения по умолчанию определены практически для всех случаев, но вы можете настроить собственную Клавиатуры панель опций.

Новые параметры пользовательского интерфейса

Есть новая опция для изменения отображения, чтобы выбранная вкладка занимала полный экран. Это полезно при использовании ZAP на маленьких экранах. Также есть возможность переключить видимость названий вкладок на выкл, чтобы еще больше сэкономить место.

Большинство списков пользовательского интерфейса также были преобразованы в таблицы, что позволяет изменять ширину столбцов и точно определять, какие столбцы будут отображаться, а также порядок сортировки таблиц.

Больше функций перенесено в дополнения

Большая часть основной функциональности была перемещена в дополнения, которые позволяют нам динамически доставлять обновления через Zap Marketplace, а не требовать новых полных выпусков.
This includes the language packs, so translations made to the ZAP UI via https://crowdin.com/project/zaproxy can be downloaded within ZAP or even automatically installed.

Новые и улучшенные правила активного и пассивного сканирования

Many of the release status active and passive scanning rules have been improved. There are new alpha and beta status rules and many rules have been promoted from alpha to beta and from beta to release status.

Прочие прочие изменения и дополнения

  • Новый параметр для остановки отдельных правил сканирования без остановки всего сканирования
  • Новая кнопка панели инструментов, которая позволяет быстро и легко записывать Zest скрипты.
  • Создана новая группа для обмена сценариями ZAP ( https://groups.google.com/group/zaproxy-scripts ).
  • Возможность создавать приложения-пауки на основе метаданных системы управления версиями (SVN и Git), предоставляемых через веб-сервер.
  • Возможность принудительного прерывания из прокси-скриптов.
